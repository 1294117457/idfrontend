# 前端架构优化方案

> 目标：在不修改任何现有功能的前提下，优化项目架构、去除无用配置、提升代码可维护性，最终形成可复用的通用前端模板。

## 二、发现的问题（按优先级排序）

### 🔴 P0 - 必须修复（影响正确性 / 构建）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 1 | **`npm` 和 `pnpm` 被错误地放入 `dependencies`** | `package.json` | 包管理器本身不应作为项目依赖安装，会增加 node_modules 体积且毫无意义。 |
| 2 | **缺少 `.env.prod` 环境文件** | 项目根目录 | `build` 脚本使用 `--mode prod`，但不存在 `.env.prod` 文件，构建时 `VITE_BASE_API` 将为 `undefined`。 |
| 3 | **缺少 ESLint 配置文件** | 项目根目录 | `package.json` 中定义了 `lint:eslint` 脚本，但没有任何 `eslint.config.*` 或 `.eslintrc.*` 文件，执行 lint 会报错或无效果。 |
| 4 | **`views/demand/manage.vue` 导入了不存在的函数** | `manage.vue` | 从 `apiProfile` 导入 `sendEmailCode`，但该函数实际在 `apiLogin.ts` 中（且命名为 `sentEmailCode`）。 |
| 5 | **路由重定向路径错误** | `router/home.ts` | `{ path: '', redirect: 'home/index' }` 应为 `{ path: '', redirect: 'index' }`（嵌套路由中 redirect 应使用相对路径或完整路径 `/home/index`）。 |

### 🟡 P1 - 应当优化（影响可维护性 / 规范性）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 6 | **`package.json` 的 `name` 为 `"test01"`** | 根目录 | 脚手架创建时的占位名称，应改为项目实际名称。 |
| 7 | **`vue-i18n` 已安装但未使用** | `package.json` / `src/` | 依赖中有 `vue-i18n`，但 `src/` 下不存在任何 i18n 配置或语言文件，`main.ts` 也没有注册。 |
| 8 | **`docx-preview`、`jszip`、`marked`、`dompurify` 使用情况不明** | `package.json` | 需确认是否被页面实际引用，若仅在已删除的功能中使用，应移除。 |
| 9 | **`@kjgl77/datav-vue3` 数据可视化库未见使用** | `package.json` | 全局搜索未发现任何组件引用，疑似残留。 |
| 10 | **`App.spec.ts` 测试已失效** | `src/__tests__/` | 期望 `App.vue` 包含 `"You did it!"`，但实际只渲染 `<router-view />`。 |
| 11 | **`src/backup.md` 放在源码目录** | `src/` | 备忘文件不应放在打包路径中。 |
| 12 | **`contact` 路由对应的视图文件内容错乱** | `views/contact/` | `index.vue` 和 `college.vue` 的实际内容是加分申请相关的 UI，与路由 meta 中声明的"BUG 提交"和"学院信息"不符。 |
| 13 | **API 层缺少统一的类型定义文件** | `src/api/components/` | `ApiResponse<T>` 在 `apiProfile.ts` 和 `apiAIagent.ts` 中各定义了一份，完全相同。 |
| 14 | **两个文件组件存在大量重复代码** | `components/fileUtil.vue` + `FileTable.vue` | 文件类型判断、预览、下载逻辑几乎完全相同（约 200 行重复），应抽取为 composable。 |
| 15 | **单文件组件过大** | `views/score/index.vue` (1259 行) | 单个组件承担过多职责，难以维护。 |

### 🟢 P2 - 建议改进（提升开发体验 / 模板通用性）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 16 | **`vite.config.ts` 缺少 `server.proxy` 配置** | `vite.config.ts` | 开发时跨域需要手动在 `.env` 中写完整后端地址，不如使用 proxy 优雅。 |
| 17 | **Element Plus 全量导入** | `main.ts` | `import ElementPlus from 'element-plus'` 是全量导入，bundle 体积较大，应改为按需导入。 |
| 18 | **`tsconfig.node.json` 包含未使用的测试框架配置** | `tsconfig.node.json` | `include` 中有 `cypress.config.*`、`nightwatch.conf.*`、`playwright.config.*`，但项目未使用这些框架。 |
| 19 | **`README.md` 为脚手架默认内容** | 根目录 | 应替换为项目实际说明。 |
| 20 | **代码中大量 `console.log` 调试输出** | 多个文件 | 生产环境应通过构建插件自动移除或使用统一的 logger 工具。 |
| 21 | **`localStorage` 直接硬编码 key** | `http.ts`、`stores/profile.ts` 等 | 应抽取为常量，避免拼写不一致（如 `access_token` vs `accessToken` 已存在的问题：`TopBar.vue` 的 logout 清除的是 `access_token`，但实际存储用的是 `accessToken`）。 |
| 22 | **路由守卫为空** | `router/index.ts` | `beforeEach` 只做了 `next()`，导入了 `useUserStore` 和 `ElMessage` 但未使用。 |

## 三、配置文件逐一审查

### 3.1 保留（有用）

| 文件 | 作用 | 是否需要修改 |
|------|------|-------------|
| `.editorconfig` | 统一编辑器格式（缩进、行尾等） | ✅ 保留，建议加上 `root = true` |
| `.env.dev` | 开发环境变量 | ✅ 保留，格式需整理（去掉多余空格） |
| `.gitattributes` | 统一行尾为 LF | ✅ 保留 |
| `.gitignore` | Git 忽略规则 | ✅ 保留，去除重复条目（`dist` 和 `dist/` 重复、`*.local` 重复） |
| `.prettierrc.json` | Prettier 格式化配置 | ✅ 保留 |
| `env.d.ts` | Vite 客户端类型声明 | ✅ 保留，建议扩展 `ImportMetaEnv` 类型 |
| `index.html` | SPA 入口 HTML | ✅ 保留，建议补充 `lang="zh-CN"` |
| `package.json` | 项目依赖和脚本 | ⚠️ 需要大幅整理 |
| `postcss.config.cjs` | PostCSS 配置（Tailwind + Autoprefixer） | ✅ 保留 |
| `tailwind.config.js` | Tailwind CSS 配置 | ✅ 保留 |
| `tsconfig.json` | TypeScript 项目引用入口 | ✅ 保留 |
| `tsconfig.app.json` | 应用代码的 TS 配置 | ✅ 保留 |
| `tsconfig.node.json` | Node 侧（构建工具）TS 配置 | ⚠️ 清理无用的 include 项 |
| `vite.config.ts` | Vite 构建配置 | ⚠️ 需要增强（proxy、auto-import 等） |

### 3.2 可删除或需评估

| 文件 | 判定 | 理由 |
|------|------|------|
| `.env.test` | ⚠️ 按需保留 | `VITE_BASE_API` 为空字符串，如果 test 环境确实需要则保留，否则删除 |

| `package-lock.json`（引用中提到） | 不存在于磁盘 | 同上 |
| `README.md` | ⚠️ 需重写 | 当前为脚手架模板内容 |

### 第二步：增强 Vite 配置（预计 20 分钟）

#### 4.2.1 完善 `vite.config.ts`

```typescript
import { fileURLToPath, URL } from 'node:url'
import { defineConfig, loadEnv } from 'vite'
import vue from '@vitejs/plugin-vue'
import vueDevTools from 'vite-plugin-vue-devtools'

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd())

  return {
    plugins: [
      vue(),
      vueDevTools(),
    ],
    resolve: {
      alias: {
        '@': fileURLToPath(new URL('./src', import.meta.url)),
      },
    },
    server: {
      port: 3000,
      proxy: {
        '/api': {
          target: env.VITE_BASE_API,
          changeOrigin: true,
        },
      },
    },
    build: {
      rollupOptions: {
        output: {
          manualChunks: {
            'element-plus': ['element-plus'],
            'vendor': ['vue', 'vue-router', 'pinia', 'axios'],
          },
        },
      },
    },
  }
})
```

**收益**：
- 开发环境请求自动代理，不再需要在 API 层拼接 `VITE_BASE_API`
- 生产构建自动分包，ElementPlus 和 vendor 包独立缓存
- 端口固定，团队协作一致

#### 4.3.2 关键变更详解

##### A. 抽取统一的 API 类型（`src/api/types.ts`）

```typescript
/** 后端统一响应格式 */
export interface ApiResponse<T = any> {
  code: number
  msg: string
  data: T
}

/** Token 相关 */
export interface TokenResponse {
  accessToken: string
  refreshToken: string
  expiresIn: number
}
```

当前 `ApiResponse` 在 `apiProfile.ts` 和 `apiAIagent.ts` 中各定义了一份，应提取到统一位置。

##### B. API 目录扁平化重命名

将 `api/components/apiXxx.ts` 改为 `api/modules/xxx.ts`：

| 旧路径 | 新路径 |
|--------|--------|
| `api/components/apiLogin.ts` | `api/modules/auth.ts` |
| `api/components/apiProfile.ts` | `api/modules/user.ts` |
| `api/components/apiScore.ts` | `api/modules/score.ts` |
| `api/components/apiFile.ts` | `api/modules/file.ts` |
| `api/components/apiDemand.ts` | `api/modules/demand.ts` |
| `api/components/apiEvaluation.ts` | `api/modules/evaluation.ts` |
| `api/components/apiAIagent.ts` | `api/modules/ai.ts` |
| `api/components/apiWechatWork.ts` | `api/modules/wechat.ts` |
| `api/components/apiFieldConfig.ts` | `api/modules/field-config.ts` |

##### C. 抽取文件操作 composable（`src/composables/useFilePreview.ts`）

`fileUtil.vue` 和 `FileTable.vue` 中有约 200 行完全相同的逻辑（文件类型判断、预览、下载），应抽取为 composable：

```typescript
// src/composables/useFilePreview.ts
export function useFilePreview() {
  const getFileExtension = (fileName: string): string => { /* ... */ }
  const canPreview = (fileName: string): boolean => { /* ... */ }
  const getPreviewType = (fileName: string) => { /* ... */ }
  const getFileTypeLabel = (fileName: string): string => { /* ... */ }
  const getFileTypeTagColor = (fileName: string): string => { /* ... */ }

  return { getFileExtension, canPreview, getPreviewType, getFileTypeLabel, getFileTypeTagColor }
}
```

##### D. 抽取 localStorage 常量（`src/constants/storage.ts`）

```typescript
export const STORAGE_KEYS = {
  ACCESS_TOKEN: 'accessToken',
  REFRESH_TOKEN: 'refreshToken',
  THEME: 'theme',
} as const
```

**现存问题**：`TopBar.vue` 中 logout 清除的是 `access_token`（下划线），但实际存储用的是 `accessToken`（驼峰），这会导致 token 无法被正确清除。统一使用常量可避免此类问题。

##### E. 修复路由重定向
